## 变更内容

### 1. 泛型模型识别逻辑修正

**原逻辑**：
- 开放泛型定义（如 `List<>`）的 `IsGenericModel` 返回 `true`

**新逻辑**：
- 开放泛型定义（如 `List<>`）的 `IsGenericModel` 返回 `false`
- 理由：开放泛型定义是定义本身，不是已构造的泛型

### 2. 泛型模型 Name 属性实现方式变更

**原实现**：
```csharp
public string Name => "List";
```

**新实现**：
```csharp
public string Name => typeof(List<>).Name;  // 返回 "List`1"
```

**适用范围**：
- `ListModel`：使用 `typeof(List<>).Name`
- `DictionaryModel`：使用 `typeof(Dictionary<,>).Name`
- 其他泛型定义模型参考此例实现

### 3. 源生成器支持补充

**新增要求**：
- 源生成代码中加入对 `bool IsGenericModel` 属性的支持
- 生成的模型类需要实现 `IsGenericModel` 属性

## 变更原因

1. **语义修正**：开放泛型定义不是"泛型模型"（已构造的泛型），而是"泛型定义模型"
2. **命名一致性**：使用 `typeof().Name` 确保与 .NET 类型系统命名保持一致
3. **源生成完整性**：确保生成的模型类也支持泛型识别功能

## 可能产生的影响

### 正面影响

1. **语义更准确**：`IsGenericModel` 的含义更清晰——仅对已构造的泛型返回 true
2. **命名一致性**：`Name` 属性与 .NET 类型系统保持一致
3. **功能完整性**：源生成的模型类也支持泛型识别

### 需要注意

1. **行为变化**：`ListModel.Instance.IsGenericModel` 现在返回 `false`
2. **名称变化**：`ListModel.Instance.Name` 现在返回 `List\`1` 而非 `List`
3. **测试更新**：相关的单元测试需要更新预期值
