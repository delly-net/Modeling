## 变更内容

### 1. 接口定义架构变更

**原方案**：所有泛型相关方法定义在 `IBaseModel` 接口
```csharp
public interface IBaseModel
{
    bool IsGenericModel { get; }
    IBaseModel GetGenericModelDefinition();
    IReadOnlyList<IBaseModel> GetGenericModels();
    IBaseModel MakeGenericModel(params IBaseModel[] models);
}
```

**新方案**：泛型相关方法（除 `IsGenericModel` 外）定义在 `IModel` 接口
```csharp
public interface IBaseModel
{
    bool IsGenericModel { get; }
}

public interface IModel : IBaseModel
{
    IModel GetGenericModelDefinition();
    IReadOnlyList<IModel> GetGenericModels();
    IModel MakeGenericModel(params IModel[] models);
}
```

### 2. 返回类型变更

- `GetGenericModelDefinition()` 返回类型从 `IBaseModel` 变更为 `IModel`
- `GetGenericModels()` 返回类型从 `IReadOnlyList<IBaseModel>` 变更为 `IReadOnlyList<IModel>`
- `MakeGenericModel()` 参数和返回值从 `IBaseModel` 变更为 `IModel`

### 3. 实现类接口实现变更

- `ListModel` 和 `DictionaryModel` 从实现 `IBaseModel` 变更为实现 `IModel`
- `ConstructedGenericModel` 从实现 `IBaseModel` 变更为实现 `IModel`
- 所有实现类需要实现 `IModel` 接口的 `GetProperties()` 和 `GetProperty()` 方法

### 4. 源生成器代码变更

生成的泛型模型类需要实现 `IModel` 接口，包含：
- 泛型相关方法（使用 `IModel` 类型）
- 属性访问方法 `GetProperties()` 和 `GetProperty()`

## 变更原因

1. **接口层次清晰**：将泛型建模能力提升到 `IModel` 层级，与属性访问能力保持一致
2. **类型安全增强**：使用 `IModel` 返回类型确保泛型模型具备属性访问能力
3. **源生成一致性**：生成的实体模型（`IModel`）与手动定义的泛型模型使用相同的接口层次

## 可能产生的影响

### 正面影响

1. **接口层次更合理**：泛型建模能力与属性访问能力都在 `IModel` 层级
2. **类型一致性**：所有泛型模型都具备属性访问能力
3. **源生成简化**：生成的代码统一实现 `IModel` 接口

### 需要注意

1. **实现复杂度增加**：泛型模型类需要实现属性访问方法
2. **属性实现**：对于没有属性的泛型类型，`GetProperties()` 返回空数组
3. **现有代码**：需要更新所有使用 `IBaseModel` 的泛型相关代码
