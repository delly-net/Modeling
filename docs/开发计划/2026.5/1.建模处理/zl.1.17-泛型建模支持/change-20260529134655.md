## 变更内容

### 1. 新增接口成员

**新增 `MakeGenericModel` 函数**：
```csharp
/// <summary>
/// 根据泛型参数创建已构造的泛型模型
/// </summary>
/// <param name="models">泛型参数对应的模型列表</param>
/// <returns>已构造的泛型模型</returns>
IBaseModel MakeGenericModel(params IBaseModel[] models);
```

### 2. 新增常见泛型对象类

- **ListModel**：`List<T>` 的建模定义类
- **DictionaryModel**：`Dictionary<TKey, TValue>` 的建模定义类
- 其他常见泛型集合类型（如需要）

### 3. 架构设计变更

**原方案**：
- 使用工厂模式 `ModelFactory.GetModel()` 获取泛型定义模型
- 泛型模型通过工厂注册和管理

**新方案**：
- 使用实例模式，如 `ListModel.Instance`
- 泛型建模通过 `MakeGenericModel()` 方法创建
- 泛型建模本身不支持源生成

### 4. 使用示例变更

**原示例**：
```csharp
var definition = listIntModel.GetGenericModelDefinition();
_genericDefinition = ModelFactory.GetModel(_genericDefinitionType);
```

**新示例**：
```csharp
// 创建已构造的泛型模型
var listInt32Model = ListModel.Instance.MakeGenericModel(Int32Model.Instance);
var dictStringIntModel = DictionaryModel.Instance.MakeGenericModel(StringModel.Instance, Int32Model.Instance);
```

## 变更原因

1. **AOT 友好性**：实例方法模式比工厂模式更符合 AOT 编译要求
2. **简化使用**：用户直接调用 `MakeGenericModel()` 即可创建泛型模型，无需工厂注册
3. **明确职责**：泛型定义模型（如 `ListModel`）负责管理自身的泛型创建逻辑
4. **源生成限制**：泛型建模本身不支持源生成，需要预定义实例

## 可能产生的影响

### 正面影响

1. **更好的 AOT 兼容性**：避免运行时工厂查找
2. **更清晰的 API**：`MakeGenericModel()` 语义明确
3. **更少的运行时开销**：无需工厂注册和查找

### 需要注意

1. **覆盖范围**：需要为常用泛型类型预定义建模类
2. **扩展性**：新增泛型类型需要添加新的建模类
3. **现有代码**：需要更新现有的泛型模型获取方式
