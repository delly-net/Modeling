## Prompt 文档理解

### 功能概述
在 `DefaultEntityModelFactory` 的构造函数中初始化所有基础类型建模，使工厂能够直接提供基础类型（如 `string`、`int`、`long`、`bool`、`double`、`decimal`、`DateTime`、`Guid`）的建模支持。

### 重要约束
- 基础类型建模在工厂构造时自动注册，使用类全称 `{Namespace}.{ClassName}` 作为键
- 保持 `DefaultEntityModelFactory._models` 字段类型为 `Dictionary<string, IEntityModel>` 不变
- 现有 `IBaseModel` 实现类（StringModel、Int32Model 等）保持不变
- 为基础类型创建新的 `IEntityModel` 适配器类

### 实现要求
1. 创建 8 个基础类型的 `IEntityModel` 适配器类
2. 在 `DefaultEntityModelFactory` 中添加基础类型静态 Type 字段
3. 实现 `RegisterBaseTypeModels()` 私有方法
4. 在构造函数中调用 `RegisterBaseTypeModels()`
5. 重构 `GetModel<T>()` 方法支持基础类型
6. 添加 `TryGetModelFromModelSets<T>()` 辅助方法
7. 编译通过，无新增错误
8. Deme 项目中添加示例代码

## 备注

### 遇到的问题及解决方案

1. **缺少 using 引用错误**
   - 问题：适配器类无法找到 StringModel、Int32Model 等
   - 解决：在所有适配器类中添加 `using Delly.Modeling.Models;`

2. **C# 7.3 兼容性问题**
   - 问题：递归模式 `var t when t ==` 在 netstandard2.0 中不支持
   - 解决：使用 if-else 语句替代 switch 表达式

3. **泛型约束限制**
   - 问题：`GetModel<T>()` 方法有 `where T : class` 约束，无法直接使用值类型
   - 解决：值类型通过类全称（如 "System.Int32"）查询，在 Demo 中正确演示

4. **Nullable 警告**
   - 问题：部分 nullable 警告在现有代码中存在
   - 解决：保持现有警告不变，专注于本次任务新增代码
