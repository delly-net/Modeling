## 变更内容

### 1. 基础建模获取逻辑优化

**变更原因**: 需求补充文档新增"基础建模获取逻辑优化"需求

**变更内容**:
- 原方案：在 `DefaultEntityModelFactory` 构造函数中通过 `RegisterBaseTypeModels()` 方法预注册所有基础类型建模到 `_models` 字典
- 新方案：不在构造函数中初始化，在 `GetModel` 方法中优先判断类型后直接返回静态实例

**技术方案调整**:
- 移除 `RegisterBaseTypeModels()` 私有方法
- 移除构造函数中对基础类型的注册调用
- `GetModel<T>()` 方法中使用 `typeof + switch case` 直接返回静态实例，不依赖字典查找
- `GetModel(string fullName)` 方法中同样优先判断基础类型全称，直接返回静态实例
- `_models` 字典不再存储基础类型建模

### 2. 实现步骤更新

**移除的步骤**:
- [ ] 实现 `RegisterBaseTypeModels()` 私有方法
- [ ] 在构造函数中调用 `RegisterBaseTypeModels()`
- [ ] 验证所有基础类型建模正确注册（改为验证静态实例返回正确）

**简化的步骤**:
- 适配器类保持不变，仍需创建 8 个基础类型的 `IEntityModel` 适配器类
- 静态 Type 字段仍需添加（用于 `typeof` 比较）
- `GetModel<T>()` 和 `GetModel(string fullName)` 方法直接返回静态实例

### 3. 代码示例更新

**实现方式变更**:
```csharp
// 原方案 - 构造函数预注册
public DefaultEntityModelFactory()
{
    _models = new Dictionary<string, IEntityModel>();
    RegisterBaseTypeModels(); // 注册所有基础类型
}

// 新方案 - GetModel 直接判断
public IEntityModel GetModel<T>()
{
    return typeof(T) switch
    {
        var t when t == _stringType => StringEntityModel.Instance,
        var t when t == _int32Type => Int32EntityModel.Instance,
        // ... 其他基础类型
        _ => TryGetModelFromModelSets<T>() ?? throw new NotSupportedException(...)
    };
}
```

## 可能产生的影响

1. **实现简化**: 移除了构造函数中的注册逻辑，代码更简洁
2. **性能优化**: 基础类型无需字典查找，直接返回静态实例
3. **内存优化**: `_models` 字典不再存储基础类型键值对
4. **行为一致**: 用户调用 `GetModel` 方法的行为保持不变
