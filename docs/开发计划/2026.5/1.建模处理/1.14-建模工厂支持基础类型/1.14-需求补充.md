## 规范要求

### XML 文档注释

所有公共成员必须包含完整的 XML 文档注释：
- 类、方法、属性使用 `///` 格式
- 参数名必须与实际参数名匹配
- 返回值描述清晰明确

### 代码风格

- 使用 `#if !NETSTANDARD2_0` 条件编译支持可空类型
- 私有成员使用 `//` 注释
- 公共成员使用 `///` 注释
- 当 `if` 语句只有一行，整体少于120个字符，则将判断语句与执行语句合并为一行
  - 如 `if (a) { b(); }`

### 命名规范

- 方法名使用 PascalCase
- 私有字段使用 _camelCase
- 辅助方法使用清晰描述性名称
