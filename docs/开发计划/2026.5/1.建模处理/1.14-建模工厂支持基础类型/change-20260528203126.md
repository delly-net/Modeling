## 变更内容

### 1. 存储方案架构变更

**变更原因**: 需求补充文档更新

**变更内容**:
- 原方案：修改 `_models` 字段类型为 `Dictionary<string, IBaseModel>`，使用现有基础类型建模
- 新方案：保持 `_models` 字段类型为 `Dictionary<string, IEntityModel>`，为每个基础类型创建新的 `IEntityModel` 适配器类

**技术方案调整**:
- 使用适配器模式，为 8 个基础类型创建 `IEntityModel` 实现类
- 适配器类内部委托给现有的 `IBaseModel` 实例
- `GetProperties()` 和 `GetIndexes()` 返回空数组
- `GetProperty(name)` 返回 null

### 2. 接口实现细节

**新增适配器类**（需新建）:
- `StringEntityModel` - String 实体建模适配器
- `Int32EntityModel` - Int32 实体建模适配器
- `Int64EntityModel` - Int64 实体建模适配器
- `BooleanEntityModel` - Boolean 实体建模适配器
- `DoubleEntityModel` - Double 实体建模适配器
- `DecimalEntityModel` - Decimal 实体建模适配器
- `DateTimeEntityModel` - DateTime 实体建模适配器
- `GuidEntityModel` - Guid 实体建模适配器

### 3. 工厂类调整

**保持不变**:
- `_models` 字段类型仍为 `Dictionary<string, IEntityModel>`
- `Add(IEntityModel)` 方法签名不变
- `GetModel(string)` 方法签名不变

**新增内容**:
- 基础类型静态 Type 字段
- `RegisterBaseTypeModels()` 私有方法
- 构造函数中调用注册方法
- `GetModel<T>()` 方法扩展支持基础类型

## 可能产生的影响

1. **新增文件**: 需要在 `Delly.Modeling/Models/EntityModels/` 目录下创建 8 个新的适配器类
2. **依赖关系**: 适配器类依赖现有的 `IBaseModel` 实现（StringModel、Int32Model 等）
3. **工厂测试**: 需要验证工厂能正确查询基础类型建模
4. **向后兼容**: 现有实体模型功能不受影响
