### 1. 修改 DefaultEntityModelFactory.cs

**变更内容**:

1. **简化构造函数** - 移除对 `RegisterBaseTypeModels()` 的调用：
   ```csharp
   public DefaultEntityModelFactory()
   {
       _models = new Dictionary<string, IEntityModel>();
       _modelSets = new List<IEntityModelSet>();
       _registeredSets = new HashSet<IEntityModelSet>();
   }
   ```

2. **删除** `RegisterBaseTypeModels()` 私有方法

3. **重构 GetModel(string fullName)** - 优先判断基础类型：
   ```csharp
   switch (fullName)
   {
       case "":
       case "System.String": return StringEntityModel.Instance;
       case "System.Int32": return Int32EntityModel.Instance;
       // ... 其他基础类型
       default: return _models.TryGetValue(fullName, out var model) ? model : null;
   }
   ```

4. **重构 GetModel<T>()** - 移除 `where T : class` 约束，使用 if-else：
   ```csharp
   public IEntityModel GetModel<T>()
   {
       var typeOfT = typeof(T);
       if (typeOfT == _stringType) return StringEntityModel.Instance;
       if (typeOfT == _int32Type) return Int32EntityModel.Instance;
       // ... 其他基础类型
       var result = TryGetModelFromModelSets<T>();
       if (result != null) return result;
       throw new NotSupportedException($"类型 {typeof(T).FullName} 未在已注册的建模集合中找到");
   }
   ```

5. **重构 TryGetModel<T>()** - 移除 `where T : class` 约束，使用 if-else

6. **重构 TryGetModelFromModelSets<T>()** - 移除 `where T : class` 约束

## 备注

### 遇到的问题及解决方案

1. **接口约束冲突**
   - 问题：修改实现类移除约束后，编译器报错接口方法仍保留约束
   - 解决：同步更新 `IEntityModelFactory` 和 `IEntityModelSet` 接口定义

2. **源生成器代码不同步**
   - 问题：源生成器生成的代码仍保留 `where T : class` 约束
   - 解决：更新 `ModelSetSourceGenerator.cs` 生成模板

3. **C# 7.3 兼容性（netstandard2.0）**
   - 问题：递归模式 `var t when t ==` 在 C# 7.3 中不支持
   - 解决：使用 if-else 语句替代 switch 表达式

### 关键设计决策

1. **不预注册基础类型**：通过 switch/if-else 直接返回静态实例，避免污染 `_models` 字典
2. **移除泛型约束**：允许值类型直接使用泛型方法，提升 API 易用性
3. **C# 7.3 兼容**：使用 if-else 而非 switch 表达式，确保 netstandard2.0 目标框架兼容
