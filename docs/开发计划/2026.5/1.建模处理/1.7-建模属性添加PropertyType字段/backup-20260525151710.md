## 功能概述

在 `IModelProperty` 接口中添加 `PropertyType` 属性，使属性模型对象能够提供其所对应的 CLR 类型信息。该属性仅在源生成阶段使用 `typeof(T)` 赋值，不影响 AOT 编译兼容性。

**重要约束**：
- 属性命名为 `PropertyType`，明确表示这是属性的类型而非 PropertyModel
- 源生成器需要为 `Modelable` / `MoTable` / `MoQuery` 特性生成兼容代码
- 只在源生成阶段添加 `Type = typeof(T)`，不影响 AOT 兼容性
- 与 `PropertyModel` 区分：`PropertyType` 返回 CLR Type，`PropertyModel` 返回建模对象

## 技术背景

### 现有实现

当前 `IModelProperty` 接口定义如下：

```csharp
namespace Delly.Modeling
{
    /// <summary>
    /// 模型对象属性接口，提供属性访问能力
    /// </summary>
    public interface IModelProperty
    {
        /// <summary>
        /// 名称
        /// </summary>
        string Name { get; }

        /// <summary>
        /// 属性建模
        /// </summary>
        IBaseModel PropertyModel { get; }

        /// <summary>
        /// 获取属性值
        /// </summary>
        /// <param name="obj">目标对象</param>
        /// <returns>属性值</returns>
#if NETSTANDARD2_0
        object GetValue(object obj);
#else
        object? GetValue(object? obj);
#endif

        /// <summary>
        /// 设置属性值
        /// </summary>
        /// <param name="obj">目标对象</param>
        /// <param name="value">属性值</param>
#if NETSTANDARD2_0
        void SetValue(object obj, object value);
#else
        void SetValue(object? obj, object? value);
#endif
    }
}
```

当前生成的属性类示例（`ModelableSourceGenerator`）：

```csharp
public class p__UserModel_Id : IModelProperty
{
    // 固定静态共享实例
    private static readonly p__UserModel_Id _instance = new p__UserModel_Id();

    /// <summary>
    /// 公共实例
    /// </summary>
    public static p__UserModel_Id Instance => _instance;

    /// <summary>
    /// 属性名称
    /// </summary>
    public string Name => nameof(User.Id);

    /// <summary>
    /// 属性建模
    /// </summary>
    public IBaseModel PropertyModel => StringModel.Instance;

    /// <summary>
    /// 获取属性值
    /// </summary>
    /// <param name="obj">目标对象</param>
    /// <returns>属性值</returns>
    public object? GetValue(object? obj)
    {
        return (obj as User)?.Id;
    }

    /// <summary>
    /// 设置属性值
    /// </summary>
    /// <param name="obj">目标对象</param>
    /// <param name="value">属性值</param>
    public void SetValue(object? obj, object? value)
    {
        if (obj is User user && value is string id)
        {
            user.Id = id;
        }
    }
}
```

### 新增需求

需要在 `IModelProperty` 接口中添加 `PropertyType` 属性，以提供属性的 CLR 类型信息。

与 `PropertyModel` 的区别：
- `PropertyType`: 返回 `Type` 对象，如 `typeof(string)`、`typeof(long)`
- `PropertyModel`: 返回 `IBaseModel` 对象，如 `StringModel.Instance`、`Int64Model.Instance`

## 实现方案

### 1. 修改 IModelProperty 接口

在 `IModelProperty` 接口中添加以下属性：

```csharp
/// <summary>
/// 属性类型信息，源生成阶段使用 typeof(T) 赋值
/// </summary>
Type PropertyType { get; }
```

### 2. 源生成器实现

需要在以下源生成器中添加 `PropertyType` 属性的生成代码：

| 源生成器 | 特性 | 生成的类 |
|:--------|:----|:--------|
| `ModelableSourceGenerator` | `ModelableAttribute` | `p__{ClassName}_{PropertyName}` |
| `ModelTableSourceGenerator` | `MoTableAttribute` | `p__{ClassName}_{PropertyName}` |
| `ModelTableSourceGenerator` | `MoQueryAttribute` | `p__{ClassName}_{PropertyName}` |

生成的 `PropertyType` 属性示例：

```csharp
// p__{ClassName}_{PropertyName} 类中
/// <summary>
/// 属性类型信息
/// </summary>
public Type PropertyType => typeof({PropertyType});
```

### 3. 手动实现类更新

如果有手动实现 `IModelProperty` 的类，需要添加 `PropertyType` 属性：

```csharp
public Type PropertyType => typeof(string);
```

## 使用示例

```csharp
// 获取属性的模型信息
var userModel = User.GetEntityModel();
var idProperty = userModel.GetProperty("Id");

// 获取属性的建模对象
var propertyModel = idProperty.PropertyModel;
Console.WriteLine($"属性建模: {propertyModel.Name}"); // 输出: StringModel

// 获取属性的 CLR 类型
var propertyType = idProperty.PropertyType;
Console.WriteLine($"属性类型: {propertyType.Name}"); // 输出: String
Console.WriteLine($"类型全称: {propertyType.FullName}"); // 输出: System.String

// 创建实例并设置值
var user = (User)userModel.CreateInstance();
idProperty.SetValue(user, "test123");

// 获取并检查类型
var value = idProperty.GetValue(user);
Console.WriteLine($"值: {value}, 类型: {value?.GetType().Name}");
```
