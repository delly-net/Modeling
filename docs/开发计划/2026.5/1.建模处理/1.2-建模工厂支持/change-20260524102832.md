# 变更记录

## 原文档摘要

原文档包含了完整的功能概述、API 设计和实现方案，但存在以下问题：
1. 包含了 Type 参数的 API 方法，与 AOT 编译要求冲突
2. 存储结构按 Type 存储，与需求决策不符
3. 线程安全设计使用锁机制，与需求决策不符
4. TODO 项未根据需求补充文档进行更新

## 变更内容

1. **移除 Type 参数 API**：删除 `GetModel(Type)`、`HasModel(Type)`、`Register(Type, IModel)` 等使用 Type 参数的方法
2. **调整存储结构**：从 `Dictionary<Type, IModel>` 改为 `Dictionary<string, IModel>` 按名称存储
3. **简化线程安全**：从锁机制改为只读模式，仅在初始化时写入
4. **更新 TODO 项**：根据需求补充文档中的决策更新所有 TODO 项
5. **新增 AOT 约束**：在功能概述中强调 AOT 编译友好性要求

## 变更原因

根据 `1.2-需求补充.md` 中的明确决策进行更新：
- 注册方式：自动注册
- 存储结构：按名称存储
- 线程安全：单线程只读模式
- AOT 编译：不使用反射和 Type

## 可能产生的影响

1. API 更加简洁，完全支持 AOT 编译
2. 线程安全逻辑简化，性能提升
3. 实现难度降低，代码更易维护
