## Prompt 文档操作理解

任务 1.2 "建模工厂支持" 的核心目标是添加建模工厂能力，用于统一管理项目中所有的实体模型对象。主要实现：

1. **接口体系**: `IEntityModelSet` 和 `IEntityModelFactory` 接口定义
2. **特性**: `MoSetAttribute` 特性标记类以启用源代码生成
3. **默认实现**: `DefaultEntityModelFactory` 标准实现
4. **源生成器**: `ModelSetSourceGenerator` 独立实现（不修改现有源生成器）
5. **静态工具**: `ModelUtils` 提供辅助方法

**重要约束**:
- 完全支持 AOT 编译，不使用反射
- 使用类全称 `{Namespace}.{ClassName}` 作为模型唯一标识
- 支持手动注册（运行时）和模块化开发
- 兼容 .NET Standard 2.0

## 文件新建/修改/删除细节

### 新建文件

1. **Delly.Modeling/IEntityModelSet.cs**
   - 实体模型集合接口
   - 包含 `GetModels()` 和 `Count` 成员

2. **Delly.Modeling/MoSetAttribute.cs**
   - 建模集合特性
   - 标记类以启用源代码生成

3. **Delly.Modeling/IEntityModelFactory.cs**
   - 实体建模工厂接口
   - 包含 `GetModel`, `GetModelByTableName`, `GetAllModels`, `GetModelsByNamespace`, `HasModel`, `Count`, `AddSet`, `Add`

4. **Delly.Modeling/DefaultEntityModelFactory.cs**
   - 默认实体建模工厂实现
   - 使用 `Dictionary<string, IEntityModel>` 存储，键为类全称

5. **Delly.Modeling.Generator/ModelSetSyntaxReceiver.cs**
   - 语法接收器，收集 `MoSet`、`MoTable`、`MoQuery` 标记的类

6. **Delly.Modeling.Generator/ModelSetSourceGenerator.cs**
   - 独立源生成器
   - 为 `[MoSet]` 类生成 `IEntityModelSet` 实现

7. **Delly.Modeling/ModelUtils.cs**
   - 静态工具类
   - 提供 `GetFullName(IEntityModel)` 方法

### 修改文件

1. **Delle.Modeling/DefaultEntityModelFactory.cs**
   - 修复兼容性问题：`_models = []` → `new Dictionary<string, IEntityModel>()`

2. **Deme/Program.cs**
   - 添加建模工厂使用示例
   - 添加模块化开发示例（ModuleA、ModuleB）
