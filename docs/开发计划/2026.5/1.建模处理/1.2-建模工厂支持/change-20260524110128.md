# 变更记录

## 原文档摘要

原文档采用 `ModelFactory` 静态类来管理所有模型，通过自动注册方式将模型注册到工厂。支持按名称、泛型类型和命名空间查询模型。

## 变更内容

1. **新增 IEntityModelSet 接口**：用于建模信息的自动收集
2. **新增 MoSetAttribute 特性**：用于标记程序集中的建模集合类，支持源生成
3. **新增 IEntityModelFactory 接口**：定义实体建模工厂的标准接口
4. **新增 DefaultEntityModelFactory 实现**：提供标准的工厂实现
5. **支持动态注册**：允许运行时添加建模集合或单个模型
6. **范围调整**：只支持 `IEntityModel`，不再支持 `IModel`

## 变更原因

根据 `1.2-需求补充.md` 中的新增开发要求进行调整：
- 需要提供更灵活的模型集合管理方式
- 需要支持程序集级别的自动收集
- 需要支持运行时动态注册（IMoSet）

## 可能产生的影响

1. 架构更加灵活，支持多种模型收集方式
2. 支持运行时动态扩展，适应更复杂的使用场景
3. 范围收窄到 IEntityModel，更专注于实体建模场景
4. 需要新增多个接口和类，实现复杂度增加
