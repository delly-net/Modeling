# 变更记录

## 原文档内容

原文档使用 `Name` 属性（表名/查询对象名称）作为模型的唯一标识和存储键。

## 变更内容

1. **修改存储键策略**：从使用 `model.Name` 改为使用类全称 `{Namespace}.{ClassName}` 作为唯一标识
2. **更新接口定义**：
   - `IEntityModelFactory.GetModel(string name)` 改为 `GetModel(string fullName)`
   - `GetModel(string name)` 变为 `GetModelByTableName(string tableName)`
3. **更新实现代码**：`Dictionary<string, IEntityModel>` 的键从 `Name` 改为 `Namespace.ClassName`
4. **更新使用示例**：获取模型时使用类全称而非表名

## 变更原因

根据 `1.2-需求补充.md` 中的开发要求：
- 建模唯一性依据可使用类全称方式进行
- 类全称生成逻辑：`{Namespace}.{ClassName}`
- `Name` 属性是表名/查询对象名称，可能与类名不同，不适合作为唯一标识

## 可能产生的影响

1. API 更加精确，避免表名冲突
2. 使用类全称更符合 AOT 编译要求
3. 与现有代码可能有兼容性问题，需要统一调整
