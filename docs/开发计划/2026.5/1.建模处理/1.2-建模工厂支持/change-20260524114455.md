# 变更记录

## 原文档内容

原文档使用 `ModelFactory` 作为静态类名称，提供统一的实体模型访问入口，包含自动初始化逻辑。

## 变更内容

1. **重命名静态类**：从 `ModelFactory` 改为 `ModelUtils`
2. **移除自动初始化**：移除静态构造函数中的自动初始化逻辑
3. **移除 Initialize 方法**：删除工厂内部的自动注册方法
4. **简化 API**：移除 `Factory` 属性，改为直接操作工厂实例
5. **调整使用方式**：改为手动实例化工厂并手动注册

## 变更原因

根据 `1.2-需求补充.md` 中的开发规范补充：
- 静态工具定义应该使用 `Utils` 结尾
- 兼容模块化开发：`IEntityModelFactory` 需要手动实例化并手动执行 AddSet 或 Add 填充建模信息

## 可能产生的影响

1. 更符合静态工具命名规范
2. 支持模块化开发，更灵活
3. 需要手动管理工厂实例和注册流程
4. 与现有自动注册方式有较大变化
