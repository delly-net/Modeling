### 2. API 设计建议

- **不要出现对AOT编译不友好的Type**
```csharp
namespace Delly.Modeling;

/// <summary>
/// 建模工厂，提供统一的模型管理能力
/// </summary>
public static class ModelFactory
{
    /// <summary>
    /// 获取指定类型的模型
    /// </summary>
    public static IModel? GetModel(Type type);

    /// <summary>
    /// 获取指定类型的模型
    /// </summary>
    public static T? GetModel<T>() where T : IModel;

    /// <summary>
    /// 获取所有模型
    /// </summary>
    public static IEnumerable<IModel> GetAllModels();

    /// <summary>
    /// 获取指定命名空间的所有模型
    /// </summary>
    public static IEnumerable<IModel> GetModelsByNamespace(string ns);

    /// <summary>
    /// 获取模型总数
    /// </summary>
    public static int Count { get; }
}
```

### 3. 开发规范

1. **AOT编译**
   - 完全AOT编译支持，不使用所有反射相关代码
   - 使用泛型或类型名称代替Type

2. **不可变集合**
   - 根据实际需求，决定返回的集合类型
     - 有数量概念 → 使用 IReadOnlyList
     - 无数量概念 → 使用 IEnumerable

3. **静态工具定义**
   - 所有静态工具定义应该使用 `Utils` 结尾

### 4. 开发要求

1. **注册功能细节**
   - 添加 IEntityModelSet 接口来进行建模信息的自动收集
   - 添加 MoSetAtrribute 特性来进行当前程序集中建模信息的自动收集
     - 添加 MoSetAtrribute 的独立源生成支持，不在原有的源生成器上修改
     - 实际使用中，通过在一个程序集中定义一个EntityModelSet类来进行MoTable与MoQuery信息的自动收集
     ```
     [MoSet]
     public partial class EntityModelSet;
     ```
   - 添加 IEntityModelFactory 接口定义来管理所有的实体建模
     - 提供建模信息的读取类操作
     - 只支持IEntityModel，不支持IModel
   - 添加 DefaultEntityModelFactory 提供标准的 IEntityModelFactory 实现
     - 通过 AddSet(IEntityModelSet modelSet)方式添加吉建模集合
     - 通过 Add(IEntityModel model)方式添加单个建模集合
   - 兼容模块化开发
     - IEntityModelFactory 需要手动实例化并手动执行AddSet或Add填充建模信息

2. **建模唯一性依据**
   - 判断建模唯一性可使用类全称方式进行
     - 类全称生成逻辑：`{Namespace}.{ClassName}`
