## 变更内容

### 1. 实现方案变更

**原方案**：
- 添加 `Dictionary<Type, string>` 用于存储类型到类全称的映射
- 在 `Add(IEntityModel model)` 方法中收集类型信息
- 在 `AddSet(IEntityModelSet modelSet)` 方法中收集类型信息
- 使用 `switch case` 匹配类型

**新方案**（基于需求补充结论）：
- 维护全局 `IEntityModelSet` 列表
- 调用 `AddSet(IEntityModelSet modelSet)` 时将 modelSet 保存到列表
- 调用 `GetModel<T>()` 时，遍历 `IEntityModelSet` 列表，依次调用每个 set 的 `TryGetModel<T>()` 方法
- `Add(IEntityModel model)` 方法不影响 `GetModel<T>` 的结果
- `AddSet` 调用只允许新增，不允许修改（已存在的同名 set 不再添加）

### 2. 待确认项确认

| 项 | 原建议 | 确认结论 |
|:---|:---|:---|
| 泛型约束 | `where T : class` | 采用 `where T : class` |
| 异常类型 | `NotSupportedException` | 采用 `NotSupportedException` |
| 类型收集方式 | 添加泛型 `Add<T>` 方法 | 不采用，使用 `AddSet` + `IEntityModelSet` 委托模式 |
| 类型映射生命周期 | 与模型字典同步 | 只允许新增，不允许修改 |

### 3. 使用示例变更

**原示例 1（泛型方式获取模型）**：删除
- 原示例依赖 `Add` 方法收集类型，现已确认 `Add` 方法不影响 `GetModel<T>` 结果

**新增示例**：依赖 `AddSet` 和 `MoSet` 特性生成的集合

```csharp
[MoSet]
public partial class EntityModelSet;

var factory = new DefaultEntityModelFactory();
var entitySet = new EntityModelSet();

// 通过建模集合注册（必需）
factory.AddSet(entitySet);

// 通过泛型获取
var userModel = factory.GetModel<User>();
var orderModel = factory.GetModel<Order>();

// 使用 TryGetModel 避免异常
var productModel = factory.TryGetModel<Product>();
if (productModel != null)
{
    Console.WriteLine(productModel.Name);
}
```

### 4. 实现步骤更新

**新增/修改步骤**：
- [ ] 在 `DefaultEntityModelFactory` 中添加 `IEntityModelSet` 列表
- [ ] 修改 `AddSet(IEntityModelSet modelSet)` 方法，将 set 保存到列表
- [ ] 修改 `AddSet(IEntityModelSet modelSet)` 方法，检查同名 set 是否已存在，存在则跳过
- [ ] 实现 `GetModel<T>()` 方法，遍历 `IEntityModelSet` 列表并调用 `TryGetModel<T>()`
- [ ] 实现 `TryGetModel<T>()` 方法，遍历 `IEntityModelSet` 列表并调用 `TryGetModel<T>()`

**删除步骤**：
- 删除添加类型映射字典的步骤
- 删除在 `Add` 方法中收集类型信息的步骤

## 变更原因

1. **简化类型收集机制**：避免维护类型到类全称的映射字典，利用 `IEntityModelSet` 已有的泛型方法实现
2. **利用现有实现**：任务 1.3 已为 `IEntityModelSet` 实现了泛型方法，可直接复用
3. **AOT 友好**：保持 `IEntityModelSet` 中的 `typeof + switch case` 实现，工厂只负责委托调用
4. **清晰职责划分**：`IEntityModelSet` 负责类型匹配，`IEntityModelFactory` 负责集合管理

## 可能产生的影响

1. **API 使用限制**：
   - 用户必须使用 `AddSet` 方法注册集合才能使用泛型方法
   - 单独使用 `Add` 方法注册的模型无法通过泛型方法获取

2. **依赖关系变更**：
   - 依赖任务 1.3 的实现（`IEntityModelSet` 泛型方法）
   - 必须与 `MoSet` 特性配合使用

3. **测试用例更新**：
   - 测试通过 `Add` 方法注册的类型不应被 `GetModel<T>` 获取
   - 测试必须先通过 `AddSet` 注册集合
