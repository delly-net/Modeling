## 变更内容

### 1. 泛型实现方式调整

**原设计**:
```csharp
public IReadOnlyList<T> GetModels<T>() where T : IEntityModel
{
    return _models.OfType<T>().ToArray();
}
```

**新设计**（来自需求补充）:
```csharp
private static Type _userType = typeof(User);

/// <summary>
/// 获取指定类型的模型
/// </summary>
/// <typeparam name="T">模型类型</typeparam>
/// <returns>指定类型的模型</returns>
public IEntityModel GetModel<T>()
{
    switch(typeof(T))
    {
        case _userType:
            return User.GetEntityModel();
        default:
            throw new NotSupportedException($"不支持的模型类型: {typeof(T)}");
    }
}
```

### 2. 核心差异点

| 项目 | 原设计 | 新设计 |
|:---|:---|:---|
| 方法名称 | `GetModels<T>()` | `GetModel<T>()` |
| 返回类型 | `IReadOnlyList<T>` | `IEntityModel` |
| 实现方式 | LINQ `OfType<T>()` | `typeof` + `switch case` |
| AOT 友好性 | 依赖 LINQ 运行时 | 完全编译时 |

## 变更原因

根据需求补充文档反馈：
1. **保持现有代码不变** - 需要确保向后兼容性
2. **泛型实现案例（需要进行评估）** - 提供了使用 `typeof` 实现类型判断的示例

## 可能产生的影响

1. **实现复杂度增加**: 源生成器需要为每个已知的模型类生成静态字段和 switch 分支
2. **AOT 友好性提升**: 避免了运行时 LINQ 反射操作，更适合 AOT 编译
3. **灵活性降低**: 使用 switch case 需要在源生成时确定所有可能的类型
4. **API 设计调整**: 从 `GetModels<T>()` 返回集合改为 `GetModel<T>()` 返回单个模型
