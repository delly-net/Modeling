# 1.1 查询对象建模支持

## 任务信息

- **任务号**: 1.1
- **任务名称**: 查询对象建模支持
- **责任人**: 郑琳
- **任务类型**: 功能开发
- **任务描述**: 添加针对查询对象的建模源生成支持

## 功能概述

为查询对象（Query Object）添加建模源生成支持，使用 `MoQueryAttribute` 特性标记查询类，通过修改现有的 `ModelTableSourceGenerator` 源代码生成器生成 `IEntityModel` 实现。查询对象与实体表共享同一套 `IEntityModel` 接口和源生成器，便于统一处理数据库查询相关的建模操作。

**重要约束**：
- `MoTableAttribute` 与 `MoQueryAttribute` 互斥，不允许同一个类同时标记两种特性
- `MoQueryAttribute` 不支持与 `MoColumnAttribute` 搭配工作，属性元数据通过自动推断获得
- 两种特性共享源生成器且生成的类命名相同

## 技术背景

### 现有实现

**MoTableAttribute** (实体表建模)：
- 标记类为数据库实体表
- 继承自 `TableAttribute`
- 通过 `ModelTableSourceGenerator` 生成 `GetEntityModel()` 方法返回 `IEntityModel`
- 生成属性访问器 `IEntityModelProperty[]`
- **必须搭配 `[MoColumn]` 特性**定义列元数据
- 支持通过 `[MoColumnIndex]` 定义索引元数据
- 生成的类：`emc__{ClassName}`, `emp__{ClassName}_{PropertyName}`

**ModelTableSourceGenerator**:
- 使用 `ModelTableSyntaxReceiver` 收集标记了 `[MoTable]` 的类
- 通过 `GetPropertyMetadata` 读取 `[MoColumn]` 特性获取属性元数据
- 只处理带有 `[MoColumn]` 特性的属性

### 新增需求

**MoQueryAttribute** (查询对象建模)：
- 标记类为查询对象
- 同样通过 `ModelTableSourceGenerator` 生成 `GetEntityModel()` 方法返回 `IEntityModel`
- 与 `MoTableAttribute` 共享同一接口体系
- **不搭配 `[MoColumn]` 特性**，属性元数据自动推断
- 两者都为 SQL 查询提供 `IEntityModel` 对象支持
- **与 MoTableAttribute 互斥**
- **生成的类命名相同**：`emc__{ClassName}`, `emp__{ClassName}_{PropertyName}`

## 实现方案

### 1. 新增特性类

在 `Delly.Modeling` 项目中创建 `MoQueryAttribute`：

```csharp

/// <summary>
/// 查询对象特性，标记查询类以启用源代码生成
/// </summary>
[AttributeUsage(AttributeTargets.Class)]
public class MoQueryAttribute : Attribute
{
}
```

### 2. 升级现有源生成器

修改 `ModelTableSourceGenerator` 同时支持 `[MoTable]` 和 `[MoQuery]` 特性：

**修改 `ModelTableSyntaxReceiver`**:
- 收集标记了 `[MoTable]` 或 `[MoQuery]` 的类
- 区分两种特性的标记

**修改 `ModelTableSourceGenerator.Execute`**:
- 在处理每个类时检查特性互斥
- 如果同时存在两种特性，报告诊断错误
- 区分两种特性的属性处理逻辑

**特性互斥检测逻辑**:

```csharp
private static bool CheckAttributeMutualExclusion(INamedTypeSymbol symbol, GeneratorExecutionContext context)
{
    var hasMoTable = symbol.GetAttributes().Any(a => a.AttributeClass?.Name == "MoTableAttribute");
    var hasMoQuery = symbol.GetAttributes().Any(a => a.AttributeClass?.Name == "MoQueryAttribute");

    if (hasMoTable && hasMoQuery)
    {
        var diagnostic = Diagnostic.Create(
            new DiagnosticDescriptor(
                "DM001",
                "Attribute Mutually Exclusive",
                "Class '{0}' cannot have both [MoTable] and [MoQuery] attributes. They are mutually exclusive.",
                "Usage",
                DiagnosticSeverity.Error,
                true),
            symbol.Locations.FirstOrDefault(),
            symbol.Name);
        context.ReportDiagnostic(diagnostic);
        return false;
    }

    return true;
}
```

**修改属性元数据处理逻辑**:

```csharp
private static PropertyMetadata? GetPropertyMetadata(IPropertySymbol property, bool isQuery)
{
    PropertyMetadata meta;

    if (isQuery)
    {
        // 查询对象：不读取 [MoColumn]，使用默认推断值
        meta = new PropertyMetadata
        {
            Symbol = property,
            ColumnName = property.Name,
            IsPrimaryKey = false,
            IsAutoIncrement = false,
            Type = InferColumnType(property.Type),
            Length = GetDefaultLength(property.Type),
            Precision = 0,
            Comment = string.Empty,
            ColumnIndexes = new List<ColumnIndexEntry>()
        };
    }
    else
    {
        // 实体表：读取 [MoColumn] 特性
        var moColumnAttr = property.GetAttributes().FirstOrDefault(a =>
            a.AttributeClass?.Name == "MoColumnAttribute");

        if (moColumnAttr == null)
            return null;

        // ... 原有的 [MoColumn] 读取逻辑 ...
    }

    return meta;
}

private static ColumnType InferColumnType(ITypeSymbol type)
{
    // 处理可空类型
    if (type is INamedTypeSymbol namedType && namedType.IsGenericType &&
        namedType.ConstructedFrom?.Name == "Nullable")
    {
        type = namedType.TypeArguments[0];
    }

    var fullTypeName = type.ToString();
    return fullTypeName switch
    {
        "string" or "System.String" => ColumnType.VARCHAR,
        "int" or "System.Int32" => ColumnType.INTEGER,
        "long" or "System.Int64" => ColumnType.LONG,
        "bool" or "System.Boolean" => ColumnType.BOOL,
        "double" or "System.Double" => ColumnType.DECIMAL,
        "decimal" or "System.Decimal" => ColumnType.DECIMAL,
        "System.DateTime" => ColumnType.TIME,
        "System.Guid" => ColumnType.UNSET,
        _ => ColumnType.VARCHAR
    };
}

private static int GetDefaultLength(ITypeSymbol type)
{
    // 处理可空类型
    if (type is INamedTypeSymbol namedType && namedType.IsGenericType &&
        namedType.ConstructedFrom?.Name == "Nullable")
    {
        type = namedType.TypeArguments[0];
    }

    var fullTypeName = type.ToString();
    return fullTypeName switch
    {
        "string" or "System.String" => 255,
        "int" or "System.Int32" => 0,
        "long" or "System.Int64" => 0,
        "bool" or "System.Boolean" => 0,
        "double" or "System.Double" => 0,
        "decimal" or "System.Decimal" => 0,
        "System.DateTime" => 0,
        _ => 255
    };
}
```

**生成内容**:
- 为两种特性生成相同结构的代码
- 生成 `GetEntityModel()` 方法返回 `emc__{ClassName}.Instance`
- 生成属性访问器类 `emp__{ClassName}_{PropertyName}` 实现 `IEntityModelProperty`
- **两种特性生成的类命名相同**：`emc__{ClassName}`, `emp__{ClassName}_{PropertyName}`
- `[MoQuery]` 使用推断的元数据，`[MoTable]` 使用 `[MoColumn]` 指定的元数据

## 使用示例

```csharp
// 查询对象 - 正确用法（无 [MoColumn]）
[MoQuery]
public partial class UserQuery
{
    public string? Username { get; set; }
    public int? MinAge { get; set; }
    public int? MaxAge { get; set; }
    public DateTime? CreatedFrom { get; set; }
}

// 实体表 - 正确用法（需要 [MoColumn]）
[MoTable("users")]
public partial class User
{
    [MoColumn(Name = "id", Key = true, AutoIncrement = true, Type = ColumnType.LONG)]
    public long Id { get; set; }

    [MoColumn(Name = "username", Type = ColumnType.VARCHAR, Length = 50)]
    public string Username { get; set; }

    [MoColumn(Name = "age", Type = ColumnType.INTEGER)]
    public int Age { get; set; }
}

// 错误用法 1 - 特性互斥
[MoTable("users")]
[MoQuery]  // 错误：两种特性互斥
public partial class InvalidClass { }

// 错误用法 2 - 查询对象搭配 [MoColumn]
[MoQuery]
public partial class InvalidQuery
{
    [MoColumn(Name = "username", Type = ColumnType.VARCHAR)]  // 错误：不支持搭配
    public string? Username { get; set; }
}

// 使用生成的代码
var model = UserQuery.GetEntityModel();
Console.WriteLine($"表名: {model.Name}");
Console.WriteLine($"类名: {model.ClassName}");

var properties = model.GetProperties();
foreach (var prop in properties)
{
    Console.WriteLine($"{prop.Name} ({prop.Type}): {prop.PropertyName}");
}
```
