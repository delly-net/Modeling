## 功能概述

在 `IBaseModel` 接口中添加 `ClassType` 属性，使模型对象能够提供其所关联的类型信息。该属性仅在源生成阶段使用 `typeof(T)` 赋值，不影响 AOT 编译兼容性。

**重要约束**：
- 属性命名为 `ClassType`，避免与关键字冲突
- 源生成器需要为 `Modelable` / `MoTable` / `MoQuery` 特性生成兼容代码
- 只在源生成阶段添加 `Type = typeof(T)`，不影响 AOT 兼容性

## 技术背景

### 现有实现

当前 `IBaseModel` 接口定义如下：

```csharp
namespace Delly.Modeling
{
    /// <summary>
    /// 模型对象接口，提供属性访问能力
    /// </summary>
    public interface IBaseModel
    {
        /// <summary>
        /// 名称
        /// </summary>
        string Name { get; }

        /// <summary>
        /// 类型全名称
        /// </summary>
        string Namespace { get; }

        /// <summary>
        /// 创建模型类型的新实例
        /// </summary>
        /// <param name="args">构造函数参数数组</param>
        /// <returns>模型类型的新实例</returns>
        object CreateInstance(params object[] args);
    }
}
```

### 新增需求

需要在 `IBaseModel` 接口中添加 `Type ClassType` 属性，以提供类型信息。

## 实现方案

### 1. 修改 IBaseModel 接口

在 `IBaseModel` 接口中添加以下属性：

```csharp
/// <summary>
/// 模型类型信息，源生成阶段使用 typeof(T) 赋值
/// </summary>
Type ClassType { get; }
```

### 2. 源生成器实现

需要在以下源生成器中添加 `ClassType` 属性的生成代码：

| 源生成器 | 特性 | 生成的类 |
|:--------|:----|:--------|
| `ModelableSourceGenerator` | `ModelableAttribute` | `emc__{ClassName}` |
| `ModelTableSourceGenerator` | `MoTableAttribute` | `emc__{ClassName}` |
| `ModelTableSourceGenerator` | `MoQueryAttribute` | `emc__{ClassName}` |

生成的 `ClassType` 属性示例：

```csharp
// emc__{ClassName} 类中
public override Type ClassType => typeof({ClassName});
```

### 3. 手动实现类更新

如果有手动实现 `IBaseModel` 的类，需要添加 `ClassType` 属性：

```csharp
public override Type ClassType => typeof(YourClass);
```

## 使用示例

```csharp
// 获取模型的类型信息
var model = Order.GetEntityModel();
Console.WriteLine($"类型: {model.ClassType.FullName}");
Console.WriteLine($"名称: {model.Name}");
Console.WriteLine($"命名空间: {model.Namespace}");

// 创建实例
var instance = (Order)model.CreateInstance(1L, "test");
```
