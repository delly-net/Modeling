# 变更记录 - 1.5 建模创建对象

## 备份信息

**原始文档备份**: [backup-20260524233404.md](./backup-20260524233404.md)

## 变更内容

### 1. 接口方法签名变更

**原始设计**：
```csharp
public interface IBaseModel
{
    object CreateInstance();
}
```

**新设计**：
```csharp
public interface IBaseModel
{
    object CreateInstance(object?[] args);
}
```

### 2. 参数化构造函数支持

**第一次返工**（已推翻）：
- 不允许使用带参数的构造函数

**第二次返工**（新设计）：
- **允许定义带参数的构造函数**
- 支持根据不同的参数数量实现多构造函数
- 多构造函数情况下，出现参数数量相同的，需要抛出异常

### 3. 源生成实现变更

**原始设计**：
```csharp
public object CreateInstance()
{
    return new User();
}
```

**新设计**：
```csharp
public object CreateInstance(object?[] args)
{
    return args.Length switch
    {
        0 => new User(),
        1 => new User((string)args[0]),
        2 => new User((string)args[0], (int)args[1]),
        _ => throw new ArgumentException($"不支持 {args.Length} 个参数的构造函数")
    };
}
```

### 4. 多构造函数处理

**源代码**：
```csharp
[Modelable]
public partial class User
{
    public User() { }
    public User(string id) { Id = id; }
    public User(string id, int age) { Id = id; Age = age; }
    public string Id { get; set; } = string.Empty;
    public int Age { get; set; }
}
```

**生成代码**：
```csharp
public object CreateInstance(object?[] args)
{
    return args.Length switch
    {
        0 => new User(),
        1 => new User((string)args[0]),
        2 => new User((string)args[0], (int)args[1]),
        _ => throw new ArgumentException($"不支持 {args.Length} 个参数的构造函数")
    };
}
```

**异常情况**（参数数量相同）：
```csharp
// 源代码 - 存在两个单参数构造函数
[Modelable]
public partial class User
{
    public User(string id) { Id = id; }  // 构造函数1
    public User(string name) { Name = name; }  // 构造函数2 - 参数数量相同
    public string Id { get; set; } = string.Empty;
    public string Name { get; set; } = string.Empty;
}
```

**生成代码（带异常）**：
```csharp
public object CreateInstance(object?[] args)
{
    return args.Length switch
    {
        0 => throw new InvalidOperationException($"存在 0 个参数的构造函数冲突"),
        1 => throw new InvalidOperationException($"存在 1 个参数的构造函数冲突，无法确定使用哪一个"),
        _ => throw new ArgumentException($"不支持 {args.Length} 个参数的构造函数")
    };
}
```

## 变更原因

根据需求补充文档的反馈（第二次返工）：
1. 推翻第一次返工中的约束，改为允许使用带参数的构造函数
2. 通过 `object?[] args` 参数实现灵活的对象创建
3. 支持多构造函数场景

## 可能产生的影响

1. **接口变更**：`IBaseModel.CreateInstance()` 方法签名从无参改为带参
2. **源生成器变更**：
   - `ModelableSourceGenerator` 需要分析构造函数并生成 `switch` 表达式
   - `ModelTableSourceGenerator` 需要分析构造函数并生成 `switch` 表达式
3. **多构造函数冲突检测**：需要检测参数数量相同的情况
4. **AOT 兼容性**：使用 `switch` 表达式处理参数，AOT 友好
5. **使用方式变化**：调用时需要传递参数数组

## 差异点总结

| 项目 | 原始设计 | 第一次返工 | 新设计 |
|:-----|:---------|:-----------|:-------|
| 接口方法 | `CreateInstance()` | `CreateInstance()` | `CreateInstance(object?[] args)` |
| 构造函数支持 | 支持无参和主构造函数 | 仅支持无参构造函数 | **支持无参和带参数构造函数** |
| 多构造函数 | 不支持 | 不支持 | 支持（需检测冲突） |
| 参数冲突 | N/A | N/A | 抛出 InvalidOperationException |
| 源生成方式 | `new User()` | `new User()` | `switch` 表达式匹配 |
| 调用方式 | `model.CreateInstance()` | `model.CreateInstance()` | `model.CreateInstance(args)` |
