# 1.5 需求补充

本文档用于记录任务 1.5-建模创建对象 的需求补充和变更。

## 初始需求

添加建模直接创建对象实例的能力。

## 补充内容

### 待补充事项

1. **实现要求**
   - 在 `IBaseModel` 中添加 `object CreateInstance()` 函数
   - 在源生成时，添加 `CreateInstance` 函数的实现
   - 同时兼容 `IModel` 与 `IEntityModel` 的源生成处理

2. **第一次返工**
   - 所有使用 `Modelable` / `MoTable` / `MoQuery` 特性的对象，不允许定义带参数的构造函数
   - Demo的编写应该建立 `功能特有Class` 再由 `Program.cs` 调用，不要将所有代码都放到 `Program.cs` 中

3. **第二次返工**
   - 推翻**第一次返工**中的需求：所有使用 `Modelable` / `MoTable` / `MoQuery` 特性的对象，不允许定义带参数的构造函数
     - 改为允许定义带参数的构造函数
   - 在 `IBaseModel` 中的 `object CreateInstance()` 函数改为 `object CreateInstance(object?[] args)`
     - 同步修改源生成器，改为支持 `CreateInstance(object?[] args)` 实现中的参数支持，并可以根据不同的参数数量实现多构造函数的支持
       - 多构造函数情况下，出现参数数量相同的，需要进行异常抛出
