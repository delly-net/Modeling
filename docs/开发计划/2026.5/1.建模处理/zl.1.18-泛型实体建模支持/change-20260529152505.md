## 主要变更内容

### 1. 新增常见泛型实体模型类

**新增内容**：
- 添加 `ListEntityModel`、`DictionaryEntityModel` 等预定义泛型定义模型
- 这些模型类提供常见泛型集合类型的实体建模支持

**实现规范**：
```csharp
private readonly static Type _listType = typeof(List<>);
public string Name => _listType.Name;
```

### 2. 工厂泛型支持

**新增内容**：
- `DefaultEntityModelFactory.GetModel<T>()` 需要支持泛型类型的获取
- 实现逻辑：
  1. 通过 `typeof(T)` 获取类型
  2. 判断类型是否为泛型
  3. 如果为泛型 → 执行泛型逻辑（获取公开泛型定义类与泛型参数类型）
  4. 如果为非泛型 → 执行现有常规逻辑

### 3. 已确认决策

| TODO 项 | 确认结果 |
|:-------|:-------|
| TODO 1: 泛型实体模型是否需要支持泛型属性的建模 | **不支持** |
| TODO 2: 泛型定义模型类是否需要单独文件存放 | 生成在 obj/Generated，添加常见泛型对象，工厂支持 |
| TODO 3: 已构造泛型实体模型是否需要缓存机制 | 在泛型定义模型中使用 **List 缓存** |

## 可能产生的影响

### 正面影响

1. **更好的泛型支持**：预定义的常见泛型实体模型类（如 `ListEntityModel`、`DictionaryEntityModel`）提供了开箱即用的泛型集合建模支持
2. **工厂统一性**：`DefaultEntityModelFactory.GetModel<T>()` 对泛型类型的支持，使得泛型和非泛型模型的获取方式统一
3. **命名一致性**：`Name` 属性使用 `typeof(List<>).Name` 方式，与 zl.1.17 中的 `ListModel` 等保持一致

### 需要注意的点

1. **源生成器适配**：需要在 `ModelTableSourceGenerator` 中为常见泛型类型生成对应的模型代码
2. **工厂逻辑复杂度**：`GetModel<T>()` 需要增加泛型类型判断逻辑
3. **缓存管理**：泛型定义模型中的 List 缓存需要考虑线程安全性（如果存在多线程访问场景）
