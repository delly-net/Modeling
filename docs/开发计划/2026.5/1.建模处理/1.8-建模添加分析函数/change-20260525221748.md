## 变更内容

### 1. 新增 ParsableAttribute 特性

**新增**：
- 添加 `ParsableAttribute` 特性，用于源生成时进行 `IParsable<T>` 实现类定位
- 实现类同时实现 `IParsable<T>` 接口并标记 `[ParsableFor(typeof(ClassName))]` 特性

**原因**：
- 解决源生成器无法通过命名约定查找实现类的问题
- 提供更灵活的命名方式

### 2. 实现类定位方式变更

**原始方案**：
- 使用 `{ClassName}Parser` 命名约定
- 源生成器硬编码 `new {ClassName}Parser()`

**新方案**：
- 使用 `ParsableAttribute` 特性标记实现类
- 源生成器通过特性找到实现类名

### 3. 多实现检测

**新增**：
- 检测到多个类实现 `IParsable<{ClassName}>` 时抛出异常

**原因**：
- 避免歧义，明确要求唯一实现

### 4. 实例对象缓存

**新增**：
- 支持 Parser 实例对象缓存
- 使用静态字段缓存单例

**原因**：
- 减少对象分配，提高性能

## 变更原因

需求补充要求使用特性标记而非命名约定来定位实现类，并提供缓存支持。

## 可能产生的影响

1. **用户代码变更**：
   - 需要添加 `[ParsableFor(typeof(ClassName))]` 特性
   - 不再强制使用 `{ClassName}Parser` 命名

2. **源生成器复杂度增加**：
   - 需要扫描类上的特性
   - 需要处理多实现检测

3. **性能提升**：
   - 通过缓存减少对象分配
