## 技术决策记录

### 为什么使用反射而非静态抽象成员？

1. **源生成器限制**：`Delly.Modeling.Generator` 必须在 `netstandard2.0` 上编译
2. **静态抽象成员要求**：需要 .NET 7.0+ 和 C# 11，与目标框架不兼容
3. **向后兼容性**：反射方案在 `netstandard2.0` 和 `net6.0` 上均可工作

### 反射方案的优势

1. **框架兼容性**：无需等待 .NET 7.0+ 普及
2. **实现灵活性**：类型可自定义静态方法签名（单参数）
3. **错误处理**：调用失败时优雅降级返回 `null`

### 反射方案的劣势

1. **性能开销**：反射调用比直接调用慢
2. **编译时检查缺失**：无法在编译时验证方法签名

## 使用示例

```csharp
#if !NETSTANDARD2_0
[Modelable]
public partial class Order : IParsable
{
    public long Id { get; set; }
    public string Name { get; set; }

    public static Order? TryParse(object? obj)
    {
        if (obj == null)
            return null;

        if (obj is Order order)
            return order;

        if (obj is long id)
            return new Order { Id = id, Name = string.Empty };

        return null;
    }
}

var orderModel = Order.GetModel();
var order = (Order)orderModel.Parse(123L);
Console.WriteLine(order.Id);  // 输出: 123
#endif
```
