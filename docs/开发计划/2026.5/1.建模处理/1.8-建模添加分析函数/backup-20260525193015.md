# 1.8 建模添加分析函数

## 任务信息

- **任务号**: 1.8
- **任务名称**: 建模添加分析函数
- **责任人**: 郑琳
- **任务类型**: 代码修改
- **任务描述**: 在 `IBaseModel` 中添加 `object Parse(object? obj)` 与 `object? TryParse(object? obj)` 函数

## 功能概述

在 `IBaseModel` 接口中添加 `Parse` 和 `TryParse` 函数，提供对象值解析能力。同时新增 `IParsable` 接口作为类型级别的可分析能力标记，源生成器在生成解析逻辑时会优先检查目标类型是否实现了 `IParsable` 接口。`Parse` 函数用于将输入对象解析为目标类型实例，解析失败时抛出异常；`TryParse` 函数提供安全的解析方式，解析失败时返回 `null`。

**重要约束**：
- 所有 `interface` 定义都不允许使用 `static abstract`
- 本项目不允许出现反射方式调用
- 接口定义使用非空类型 `object`，实现类内部可返回 `null`
- `Parse` 函数解析失败时应抛出异常
- `TryParse` 函数解析失败时应返回 `null`，不应抛出异常

## 技术背景

### 现有实现

当前 `IBaseModel` 接口定义如下：

```csharp
namespace Delly.Modeling
{
    /// <summary>
    /// 模型对象接口，提供属性访问能力
    /// </summary>
    public interface IBaseModel
    {
        /// <summary>
        /// 名称
        /// </summary>
        string Name { get; }

        /// <summary>
        /// 类型全名称
        /// </summary>
        string Namespace { get; }

        /// <summary>
        /// 创建模型类型的新实例
        /// </summary>
        /// <param name="args">构造函数参数数组</param>
        /// <returns>模型类型的新实例</returns>
        object CreateInstance(params object[] args);

        /// <summary>
        /// 模型类型信息，源生成阶段使用 typeof(T) 赋值
        /// </summary>
        Type ClassType { get; }
    }
}
```

现有的基础 Model 对象（如 `StringModel`、`Int32Model`）手动实现了 `IBaseModel` 接口：

```csharp
public sealed class StringModel : IBaseModel
{
    private static readonly StringModel _instance = new StringModel();

    public static StringModel Instance => _instance;

    public string Name => nameof(String);
    public string Namespace => nameof(System);
    public Type ClassType => typeof(string);
    public object CreateInstance(params object[] args) => args == null || args.Length == 0 ? string.Empty : (args[0] as string) ?? string.Empty;
}
```

### 新增需求

1. **新增 IParsable 接口**：类型级别的可分析能力标记（不使用 static abstract）
2. **在 IBaseModel 中添加解析函数**：`Parse` 和 `TryParse`
3. **源生成逻辑更新**：支持 `IParsable` 接口检测
4. **基础 Model 对象更新**：添加独立的 `Parse` 和 `TryParse` 函数

## 实现方案

### 1. 新增 IParsable 接口

**注意**：不使用 `static abstract`，使用非空类型 `object`。类型自身提供静态方法，但接口不约束它。

```csharp
namespace Delly.Modeling
{
    /// <summary>
    /// 可分析类型接口，标记类型支持自定义解析逻辑
    /// </summary>
    /// <remarks>
    /// 类型应实现此接口并提供静态的 TryParse 方法用于解析逻辑。
    /// 源生成器会检测类型是否实现此接口，并调用其静态 TryParse 方法。
    /// </remarks>
    public interface IParsable
    {
        // 标记接口，无需定义成员
        // 类型应提供静态方法：public static object TryParse(object obj)
    }
}
```

### 2. 修改 IBaseModel 接口

**注意**：接口定义使用非空类型 `object`。

```csharp
/// <summary>
/// 将输入对象解析为目标类型实例
/// </summary>
/// <param name="obj">输入对象</param>
/// <returns>目标类型实例</returns>
/// <exception cref="ArgumentException">当对象无法转换为目标类型时抛出</exception>
object Parse(object obj);

/// <summary>
/// 尝试将输入对象解析为目标类型实例
/// </summary>
/// <param name="obj">输入对象</param>
/// <returns>目标类型实例，解析失败时返回 null</returns>
object TryParse(object obj);
```

### 3. 源生成器实现

需要在以下源生成器中添加 `Parse` 和 `TryParse` 函数的生成代码：

| 源生成器 | 特性 | 生成的类 |
|:--------|:----|:--------|
| `ModelableSourceGenerator` | `ModelableAttribute` | `emc__{ClassName}` |
| `ModelTableSourceGenerator` | `MoTableAttribute` | `emc__{ClassName}` |
| `ModelTableSourceGenerator` | `MoQueryAttribute` | `emc__{ClassName}` |

**生成的 `TryParse` 函数逻辑**：

```csharp
public override object TryParse(object obj)
{
    // 检查目标类型是否实现了 IParsable 接口
    // 使用源生成时确定的类全称，避免运行时反射
    if (typeof({ClassName}).IsAssignableTo(typeof(IParsable)))
    {
        // 调用类型的静态 TryParse 方法
        // 由于类型已知，可以硬编码调用
        return {ClassName}.TryParse(obj);
    }

    // 没有实现 IParsable → 抛出异常
    throw new ArgumentException($"Type {ClassName} does not implement IParsable interface");
}
```

**生成的 `Parse` 函数逻辑**：

```csharp
public override object Parse(object obj)
{
    // 先调用 TryParse 函数
    var result = TryParse(obj);

    // 判断是否为空，为空则抛出异常
    if (result == null)
        throw new ArgumentException($"Cannot convert {(obj != null ? obj.GetType().Name : "null")} to {ClassName}");

    return result;
}
```

### 4. 基础 Model 对象更新

为所有基础 Model 对象（`StringModel`、`Int32Model`、`Int64Model`、`BooleanModel`、`DecimalModel`、`DoubleModel`、`DateTimeModel`、`GuidModel`）添加 `Parse` 和 `TryParse` 函数：

```csharp
public sealed class Int32Model : IBaseModel
{
    // ... 现有代码 ...

    /// <summary>
    /// 将输入对象解析为 Int32 实例
    /// </summary>
    /// <param name="obj">输入对象</param>
    /// <returns>Int32 实例</returns>
    public object Parse(object obj)
    {
        var result = TryParse(obj);
        if (result == null)
            throw new ArgumentException($"Cannot convert {(obj != null ? obj.GetType().Name : "null")} to Int32");
        return result;
    }

    /// <summary>
    /// 尝试将输入对象解析为 Int32 实例
    /// </summary>
    /// <param name="obj">输入对象</param>
    /// <returns>Int32 实例，解析失败时返回 null</returns>
    public object TryParse(object obj)
    {
        if (obj == null)
            return null;

        if (obj is int value)
            return value;

        if (obj is string str && int.TryParse(str, out var intVal))
            return intVal;

        if (obj is long longVal && longVal >= int.MinValue && longVal <= int.MaxValue)
            return (int)longVal;

        return null;
    }
}
```

### 5. 支持的解析场景

**基础类型解析**（由基础 Model 对象处理）：

| 目标类型 | 支持的源类型 | 处理方式 |
|:--------|:------------|:--------|
| int | int | 直接返回 |
| int | string | 使用 int.TryParse |
| int | long | 范围检查后转换 |
| string | string | 直接返回 |
| bool | bool | 直接返回 |
| bool | string | 使用 bool.TryParse |
| TODO | TODO | TODO |

**自定义类型解析**（由 IParsable 接口处理）：
- 类型自身实现解析逻辑，提供静态 `TryParse` 方法
- 源生成器硬编码调用类型的静态 `TryParse` 方法

## 使用示例

```csharp
// 使用基础 Model 对象的解析函数
var intModel = Int32Model.Instance;

// Parse - 解析成功
int value1 = (int)intModel.Parse("123");
Console.WriteLine(value1);  // 输出: 123

// Parse - 解析失败抛出异常
try
{
    intModel.Parse("invalid");
}
catch (ArgumentException ex)
{
    Console.WriteLine(ex.Message);  // 输出: Cannot convert String to Int32
}

// TryParse - 解析成功
object result1 = intModel.TryParse("456");
if (result1 != null)
{
    Console.WriteLine(result1);  // 输出: 456
}

// TryParse - 解析失败返回 null
object result2 = intModel.TryParse("invalid");
Console.WriteLine(result2);  // 输出: (空/无值)

// 使用源生成的解析函数（针对自定义类型）
[Modelable]
public partial class Order : IParsable
{
    public long Id { get; set; }
    public string Name { get; set; }

    public static object TryParse(object obj)
    {
        if (obj == null)
            return null;

        if (obj is Order order)
            return order;

        if (obj is long id)
            return new Order { Id = id, Name = string.Empty };

        // TODO: 确认其他解析场景

        return null;
    }
}

var orderModel = Order.GetEntityModel();
var order = (Order)orderModel.Parse(123L);
Console.WriteLine(order.Id);  // 输出: 123
```
