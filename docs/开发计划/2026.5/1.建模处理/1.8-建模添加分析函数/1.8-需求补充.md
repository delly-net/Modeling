# 1.8 需求补充

本文档用于记录任务 1.8 的需求补充信息。

## 补充内容

1. **可分析能力支持**
  - 添加 `IParsable<T>` 可分析接口
    - 定义 `object? TryParse(object? obj)` 函数
      - .Net Standard 2.0 不支持Nullable类型，可定义为 `object TryParse(object obj)`
        - 此项为项目默认规范，所有函数定义时都需要默认考虑不同.net版本的兼容性
        - 使用 `#if` 为不同 .net 版本代码提供支持
    - 所有 `interface` 定义都不允许使用 `static abstract`
    - 本项目不允许出现反射方式调用！！！
    - 接口函数一定要定义，影响后续实现！！！
  - 添加 `ParsableAttribute` 特性，用于源生成时进行 `IParsable<T>` 实现类定位
    - `ParsableAttribute` 特性无需添加参数，根据 `IParsable<T>` 的 T 实现类来定义目标

2. **源生成逻辑**
  - 在处理 `Parse` 源生成时，采取先调用 `TryParse` 函数，再判断是否为空抛出为空异常
  - 在处理 `TryParse` 源生成时，判断 程序集中 是否有对象实现了 `IParsable<T>` 接口
    - 有实现 → 根据实现对象名称通过 `new {对象名}()` 方式实例化 实现对象 并调用 `TryParse` 方法
    - 没有实现 → 抛出异常

3. **基础Model对象处理**
  - 为基础 Model 对象添加独立的 `TryParse` 及 `Parse` 函数

4. **代码规范**
  - `if` 语句规范：`if` 语句判断后必须使用大括号包裹语句，只有一行代码也需要
