## 变更内容

### 1. 新增条件编译方案

- 在**重要约束**中新增：
  - `.NET Standard 2.0` 不支持 `Nullable` 类型，需要使用条件编译 `#if` 为不同 .NET 版本提供支持
  - 此项为项目默认规范，所有函数定义时都需要默认考虑不同 .NET 版本的兼容性

### 2. 更新接口定义（使用条件编译）

- **IParsable 接口**：
  - 原方案：仅使用非空类型 `object`
  - 新方案：使用条件编译，.NET Standard 2.0 使用 `object`，.NET 6.0 及以上使用 `object?`

```csharp
#if !NETSTANDARD2_0
#nullable enable
#endif

public interface IParsable
{
#if !NETSTANDARD2_0
    static abstract object? TryParse(object? obj);
#else
    static abstract object TryParse(object obj);
#endif
}
```

- **IBaseModel 接口**：
  - 原方案：仅使用非空类型 `object`
  - 新方案：使用条件编译，不同框架使用不同类型定义

### 3. 更新源生成逻辑（使用条件编译）

- **TryParse 函数逻辑变更**：
  - 原 .NET 6.0 方案：无 IParsable 实现时抛出异常
  - 原 .NET Standard 2.0 方案：直接抛出异常
  - 新方案：.NET 6.0 正常支持，.NET Standard 2.0 抛出 `NotImplementedException`

```csharp
#if !NETSTANDARD2_0
// 判断目标类型是否实现了 IParsable 接口
if (typeof({ClassName}).IsAssignableTo(typeof(IParsable)))
{
    // 有实现 → 直接调用 IParsable.TryParse 方法
    return {ClassName}.TryParse(obj);
}

// 没有实现 → 抛出异常
throw new ArgumentException($"Type {ClassName} does not implement IParsable interface");
#else
// .NET Standard 2.0 不支持静态接口成员，直接抛出异常
// TODO: 确认 .NET Standard 2.0 的替代方案
throw new NotImplementedException("IParsable is not supported in .NET Standard 2.0");
#endif
```

根据 [需求补充.md](./1.8-需求补充.md) 中的补充内容：
1. `.Net Standard 不支持Nullable类型，可定义为 object TryParse(object obj)`

3. **新增**：使用 `#if` 为不同 .net 版本代码提供支持

## 可能产生的影响

### 正面影响
1. **跨框架兼容性**：使用条件编译确保代码在 .NET Standard 2.0 和 .NET 6.0 上都能正常运行
2. **类型安全**：.NET 6.0 及以上可以使用可空类型 `object?`，提供更好的类型安全
3. **代码复用**：同一源代码支持多个框架，减少维护成本
4. **功能降级**：.NET Standard 2.0 提供明确的 `NotImplementedException`，避免隐藏的问题

### 潜在风险
1. **条件编译复杂度**：源生成器需要生成包含条件编译指令的代码，增加了实现复杂度
2. **静态接口成员限制**：.NET Standard 2.0 仍然不支持静态接口成员，需要替代方案
   - TODO: 确认 .NET Standard 2.0 的 IParsable 替代方案
3. **测试覆盖**：需要在两个框架上都进行测试，增加测试工作量

### 架构约束更新
- 更新项目级约束：**使用 `#if` 为不同 .net 版本代码提供支持**
- 建议在代码规范中添加条件编译使用的最佳实践
