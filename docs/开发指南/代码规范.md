# 代码规范

## 文档注释规范

### 公共成员

所有公共类、方法、属性和接口必须使用 XML 文档注释：

```csharp
/// <summary>
/// 获取指定属性的值
/// </summary>
/// <param name="model">模型实例</param>
/// <param name="instance">对象实例</param>
/// <param name="name">属性名称</param>
/// <returns>属性值</returns>
public static object? GetProperty(Model model, object instance, string name)
```

### 私有成员

仅使用 `//` 注释（注意空格）：

```csharp
// 缓存属性信息
private PropertyInfo[]? _properties;
```

### 参数匹配规则

XML `param` 名称必须与实际参数名称完全匹配。

## 代码风格规范

### if 语句规范

`if` 语句判断后必须使用大括号包裹语句，即使只有一行代码：

```csharp
// ✅ 正确
if (obj == null)
{
    return null;
}

// ❌ 错误
if (obj == null)
    return null;
```

**if 语句单行规范**：当 `if` 语句只有一行，整体少于120个字符，则将判断语句与执行语句合并为一行：

```csharp
// ✅ 正确 - 单行 if 语句
if (a) { b(); }

// ✅ 正确 - 多行 if 语句（超过120字符）
if (veryLongConditionName && anotherVeryLongCondition)
{
    ExecuteMethod();
}
```

## 可空注解

在接口和公共 API 中使用 `object?` 格式：

```csharp
public object? GetProperty(string name);
```

### 项目级 Nullable 配置规范

**适用范围**：所有 `*.csproj` 项目文件。

**实现方式**：在项目文件的 `<PropertyGroup>` 中配置 `<Nullable>enable</Nullable>`，而非在代码文件中使用 `#nullable enable` 指令。

**csproj 配置示例（多目标框架项目）**：
```xml
<!-- 仅给支持的框架开启 Nullable -->
<PropertyGroup Condition="'$(TargetFramework)'!='netstandard2.0'">
	<Nullable>enable</Nullable>
</PropertyGroup>
```

**csproj 配置示例（纯 .NET Standard 2.0 项目）**：
```xml
<PropertyGroup>
  <TargetFramework>netstandard2.0</TargetFramework>
  <!-- 不需要配置 Nullable -->
</PropertyGroup>
```

**关键点**：
1. 使用 `<Nullable>enable</Nullable>` 在项目级别统一管理可空引用类型配置
2. 移除代码文件中所有的 `#nullable enable` 和 `#nullable disable` 指令
3. `<Nullable>enable</Nullable>` 对 netstandard2.0 目标框架会被编译器忽略，不影响兼容性
4. 方法签名中的条件编译 `#if !NETSTANDARD2_0` 仍用于区分不同框架的类型定义
5. **纯 .NET Standard 2.0 项目不需要配置 `<Nullable>enable</Nullable>`**，保持项目文件简洁

**源生成器代码模板特殊处理**：
- 源生成器代码本身移除 `#nullable enable` 指令
- 源生成器生成的代码模板中保留 `#nullable enable` 指令
- 确保生成代码在各种目标项目中都具有正确的 nullable 行为

### Models/*.cs 可空类型支持规范

**适用范围**：`Delly.Modeling/Models/` 目录下的 Model 类文件。

**实现方式**：使用条件编译区分不同 .NET 版本，支持可空类型：

```csharp
#if !NETSTANDARD2_0
#nullable enable
#endif

public sealed class Int32Model : IBaseModel
{
#if !NETSTANDARD2_0
    /// <summary>
    /// 尝试将输入对象解析为 Int32 实例
    /// </summary>
    /// <param name="obj">输入对象</param>
    /// <returns>Int32 实例，解析失败时返回 null</returns>
    public object? TryParse(object? obj)
#else
    /// <summary>
    /// 尝试将输入对象解析为 Int32 实例
    /// </summary>
    /// <param name="obj">输入对象</param>
    /// <returns>Int32 实例，解析失败时返回 null</returns>
    public object TryParse(object obj)
#endif
    {
        if (obj == null)
            return null;

        if (obj is int value)
            return value;

        return null;
    }
}
```

**关键点**：
1. 文件顶部使用 `#if !NETSTANDARD2_0` 包裹 `#nullable enable`
2. 方法签名根据目标框架使用不同类型（`object` vs `object?`）
3. 实现内部可以返回 `null`（符合接口定义要求）

## 命名约定

- 类名：PascalCase
- 方法名：PascalCase
- 属性名：PascalCase
- 私有字段：_camelCase

## 源代码生成规范

- 生成代码输出到 `obj/Generated` 目录
- 生成的类使用 `partial` 关键字
- 为生成的扩展方法提供完整的 XML 文档注释
