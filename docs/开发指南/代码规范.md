# 代码规范

## 文档注释规范

### 公共成员

所有公共类、方法、属性和接口必须使用 XML 文档注释：

```csharp
/// <summary>
/// 获取指定属性的值
/// </summary>
/// <param name="model">模型实例</param>
/// <param name="instance">对象实例</param>
/// <param name="name">属性名称</param>
/// <returns>属性值</returns>
public static object? GetProperty(Model model, object instance, string name)
```

### 私有成员

仅使用 `//` 注释（注意空格）：

```csharp
// 缓存属性信息
private PropertyInfo[]? _properties;
```

### 参数匹配规则

XML `param` 名称必须与实际参数名称完全匹配。

## 代码风格规范

### if 语句规范

`if` 语句判断后必须使用大括号包裹语句，即使只有一行代码：

```csharp
// ✅ 正确
if (obj == null)
{
    return null;
}

// ❌ 错误
if (obj == null)
    return null;
```

## 可空注解

在接口和公共 API 中使用 `object?` 格式：

```csharp
public object? GetProperty(string name);
```

## 命名约定

- 类名：PascalCase
- 方法名：PascalCase
- 属性名：PascalCase
- 私有字段：_camelCase

## 源代码生成规范

- 生成代码输出到 `obj/Generated` 目录
- 生成的类使用 `partial` 关键字
- 为生成的扩展方法提供完整的 XML 文档注释
